- 数据监督改为 `[B,8]` labels/masks。跨 Pick→Move、Move→Place 等边界的 chunk 保留训练，不再因为整个 chunk 不稳定而丢弃；只有 unknown/padding timestep 被屏蔽。
- residual flow 只进行一次 shared solve；六个 motor experts 共享 trunk，只使用 per-token adapter/velocity heads。禁止独立采样六个 full residual chunks 后拼接。
- Stage 2 使用 oracle per-token gates warm-start；Stage 3 使用 straight-through Gumbel-Softmax 联合训练；部署使用 hard per-token Top-1。
- 推理保留 8 步预测，但只执行当前 skill 的连续前缀，最多 3 步。例如 `[Pick, Pick, Pick, Move, Move, Place, Place, Place]` 只执行前三个 Pick 动作，然后重新观测并重算完整 chunk。
- 本地 OpenVLA-OFT 的 LIBERO `NUM_ACTIONS_CHUNK=8` 与 prediction horizon 一致，但现有 evaluator action queue 默认一次消费模型返回的完整列表；实施时需要 variable-prefix adapter，确保 1～3 步队列耗尽后立即重查且不残留旧后缀。
- `null_finish` 的 solver state 必须通过 ST `motor_gate=sum(gates[:6])` 从零初始化并在每个 solver step 保持零；仅将 velocity 设零不足以消除 Gaussian residual 初值，也不能用 detached boolean clamp 切断 joint route 梯度。
